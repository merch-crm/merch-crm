Этот воркфлоу описывает процесс, которому должен следовать Antigravity при задаче "Деплой" или "Пуш в гитхаб".

## Алгоритм работы

### 1. Локальная проверка
Запустить команду:
```bash
./scripts/predeploy.sh
```
- Если возникли ошибки (типизация, линтер, тесты, билд) — **автоматически исправить их** и перезапустить проверку.
- Не переходить к следующему шагу, пока локальная проверка не пройдет успешно (✅).

### 2. Коммит и Пуш
После успешной проверки:
1. Выполнить `git add .`
2. Выполнить `git commit -m "..."` (описать изменения кратко и понятно).
3. Выполнить `git push origin main`.
4. Получить SHA коммита: `git rev-parse HEAD`.

### 3. Мониторинг GitHub Actions
Запустить скрипт мониторинга (исходя из пункта 2, подставив SHA-хэш коммита):
```bash
./scripts/monitor-deploy.sh <SHA>
```
Скрипт использует API GitHub для мониторинга процесса. Он завершится с `exit 0` в случае успеха или `exit 1` с выводом логов в случае ошибки.

### 4. Авто-исправление
- Если сборка на GitHub Actions упала:
  1. Проанализировать полученный лог.
  2. Применить исправления в коде.
  3. Вернуться к **шагу 1**.
- Если сборка прошла успешно (Success):
  1. Сообщить пользователю об успешном деплое.
  2. Предоставить ссылку на коммит.
