# Технические и Дизайнерские стандарты MerchCRM

Этот документ является **единым источником истины** для разработки и дизайна проекта MerchCRM.

> **ВАЖНО**: Общение ИИ (Antigravity) с пользователем, комментарии к коду и описание изменений должны всегда осуществляться на **русском языке**.

---

## 0. Технологический стек
- **Framework**: Next.js 16.1.6 (App Router + Turbopack)
- **Styling**: Tailwind CSS v4 (Engine: Oxide)
- **Database**: PostgreSQL (Drizzle ORM)
- **Authentication**: Better Auth (Redis-based sessions)

---

## 1. Технические стандарты разработки

### 1.1 Бизнес-логика и Архитектура
- **Domain Services**: Вся переиспользуемая бизнес-логика должна находиться в `lib/services/`.
- **Server Actions**: Выступают в роли тонких прослоек (валидация + вызов сервиса). **Обязательно** использовать `createSafeAction`.
- **Drizzle ORM**: Используйте `db.query` для чтения и билдеры для записи. Запрет на "сырой" SQL и интерполяцию строк.

### 1.2 Работа с БД
- **Индексы**: Обязательно для всех Foreign Keys (`xxx_id`), статусов (`status`, `isArchived`) и дат (`createdAt`).
- **Транзакции**: Любое изменение данных в нескольких таблицах должно быть внутри `db.transaction`.
- **Атомарность**: Используйте SQL-выражения (`sql`${inventoryItems.quantity} - 1`) для обновления счетчиков.

### 1.3 TypeScript и Типизация
- **Запрет `any`**: СТРОГО ЗАПРЕЩЕНО использовать тип `any`. Используйте `unknown` или Generic-типы.
- **Централизованные типы**: Все типы данных импортировать из `@/lib/types`.

## 2. Дизайнерские и UX стандарты

### 2.1 Основы (Foundations)
- **Сетка 4px**: Шаг отступов 4px.
- **Компактность**: Стандартные отступы — `gap-3` (12px) или `space-y-3`. Избегайте `gap-6/8`.
- **Скругления (Radius)**:
    - **XL (16px)**: Крупные блоки, таблицы.
    - **L (12px)**: Карточки, диалоги.
    - **S (6px)**: Кнопки, инпуты.

### 2.2 Язык и локализация
- **Интерфейс**: Только русский язык.
- **Склонения**: Обязательное использование `@/lib/pluralize` (`pluralize`, `sentence`). Никаких «заказ(ов)».
- **Регистр**: **Запрещен `uppercase`** в заголовках и кнопках.

### 2.3 Обязательные компоненты
AI обязан использовать готовые компоненты из `@/components/ui/` вместо написания кастомного кода:
- `ResponsiveModal` — для всех диалогов.
- `DataTable` — для таблиц (с поддержкой массового выбора).
- `ResponsiveDataView` — для адаптивности (таблица/карточки).
- `SubmitButton` — для всех форм.
- `useToast` — для уведомлений.

## 3. Рабочие процессы (Workflows)
- **Тестирование**: Перед деплоем обязателен запуск тестов (`npm run test:all`).
- **WebP**: Автоматическая конвертация всех изображений в формат `.webp` при загрузке.
- **Логирование**: Все важные действия персонала должны записываться в `audit_logs`.
