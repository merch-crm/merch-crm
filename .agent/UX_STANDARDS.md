# Дизайнерские и UX стандарты MerchCRM

Этот документ содержит правила и рекомендации по проектированию интерфейсов для MerchCRM.

## 1. Основы (Foundations)
- **Сетка 4px**: Шаг отступов 4px.
- **Компактность**: Стандартные отступы — `gap-3` (12px) или `space-y-3`. Избегайте `gap-6/8`.
- **Скругления (Radius)**:
    - **XL (16px)**: Крупные блоки, таблицы.
    - **L (12px)**: Карточки, диалоги.
    - **S (6px)**: Кнопки, инпуты.

## 2. Язык и локализация
- **Интерфейс**: Только русский язык.
- **Склонения**: Обязательное использование `@/lib/pluralize` (`pluralize`, `sentence`). Никаких «заказ(ов)».
- **Регистр**: **Запрещен `uppercase`** в заголовках и кнопках.

## 3. Обязательные компоненты
AI обязан использовать готовые компоненты из `@/components/ui/` вместо написания кастомного кода:
- `ResponsiveModal` — для всех диалогов.
- `DataTable` — для таблиц (с поддержкой массового выбора).
- `ResponsiveDataView` — для адаптивности (таблица/карточки).
- `SubmitButton` — для всех форм.
- `useToast` — для уведомлений.

## 4. Визуальные эффекты и UX
- **WebP**: Автоматическая конвертация всех изображений в формат `.webp` при загрузке.
- **Индикация**: Всегда показывать состояние загрузки (`pending`) и успеха/ошибки.
- **Адаптивность**: Интерфейс должен корректно отображаться на мобильных устройствах через `ResponsiveDataView` и `ResponsiveModal`.
