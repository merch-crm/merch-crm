# 🧬 Тестирование в MerchCRM

Этот раздел содержит документацию, реестры и результаты тестирования системы. Мы придерживаемся пирамиды тестирования для обеспечения стабильности и качества кода.

## 📂 Разделы

### 1. [[01-Unit-Тесты/Unit-Тесты|Unit-Тесты]] (Vite/Vitest)
Проверка изолированной бизнес-логики, форматтеров, хелперов и хуков. 
*Локация:* `__tests__/unit`, `lib/__tests__`.

### 2. [[02-Интеграционные-Тесты/Интеграционные-Тесты|Интеграционные-Тесты]]
Тестирование взаимодействия модулей, серверных экшенов (Actions) и API.
*Локация:* `__tests__/integration`, `tests/clients`.

### 3. [[03-E2E-Тесты/E2E-Тесты|E2E-Тесты]] (Playwright)
Полные пользовательские сценарии: от авторизации до создания заказа и отгрузки со склада.
*Локация:* `tests/dashboard`, `tests/admin`.

### 4. [[04-Компонентные-Тесты/Компонентные-Тесты|Компонентные-Тесты]]
Тестирование UI-компонентов в изоляции (SubmitButton, Select, Pagination).
*Локация:* `components/ui`.

---

## 🚀 Команды для запуска

- `npm test` — запуск всех Vitest тестов.
- `npx playwright test` — запуск E2E сценариев.
- `npm run test:ui` — графический интерфейс для Vitest.
