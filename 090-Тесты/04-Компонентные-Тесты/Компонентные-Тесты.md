# 🧬 Компонентные Тесты

Стратегия и регламент тестирования UI-компонентов MerchCRM.

## Стек
- **Vitest**: Тест-раннер.
- **React Testing Library (RTL)**: Основной инструмент проверки DOM.
- **MSW (Mock Service Worker)**: Для перехвата запросов к API.

## Основные требования
1. **Accessibility**: Проверка через роли (getByRole) — приоритет #1.
2. **User Flows**: Тестируем не пропсы, а взаимодействие пользователя (клик, ввод).
3. **Server Actions Hooks**: Мокирование хуков `useAction` из `next-safe-action`.

## Пример
```typescript
it('отображает ошибку при пустом вводе в форму заказа', async () => {
  render(<OrderForm />);
  const submit = screen.getByRole('button', { name: /создать/i });
  await userEvent.click(submit);
  expect(await screen.findByText(/обязательное поле/i)).toBeInTheDocument();
});
```
