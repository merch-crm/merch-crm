# Аналитика и Отчетность

## 1. Описание (Goal)
Модуль «Аналитика» является слоем агрегации данных для руководства и руководителей отделов. Он преобразует операционную информацию из других модулей (Заказы, Финансы, Производство) в наглядные графики, таблицы и KPI для принятия управленческих решений.

## 2. Источники данных (Data Sources)
Модуль аналитики не имеет собственных тяжелых таблиц, а работает как проекция существующих данных:
- **Заказы:** Динамика продаж, конверсия, средний чек.
- **Производство:** Время выполнения (Lead Time), загрузка линий, процент брака.
- **Клиенты:** LTV, удержание (Retention), география заказов.
- **Финансы:** P&L (прибыли и убытки), маржинальность отдельных продуктов.

## 3. Требования (Requirements)
- [x] Интерактивный дашборд с ключевыми показателями.
- [x] Фильтрация данных по периодам (день, неделя, месяц, год, произвольно).
- [x] Сравнение показателей с предыдущими периодами.
- [ ] Экспорт отчетов в PDF/Excel.
- [ ] Система прогнозирования спроса на основе исторических данных.

## 4. Техническая реализация (Implementation)
> Стандарт: [[010-Стандарты/Actions|Server Actions v3.0]]

**Файлы:**
- **Интерфейс:**
  - `app/(main)/dashboard/analytics` — Основной раздел с графиками (использует Recharts или Tremor).
- **Логика агрегации:**
  - Реализуется через специализированные Server Actions, выполняющие сложные SQL-запросы (GROUP BY, SUM, COUNT) через Drizzle ORM.
