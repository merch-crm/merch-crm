# Системное Ядро и Wiki

## 1. Описание (Goal)
Модуль «Система» отвечает за базовые инфраструктурные функции приложения: хранение файлов, внутреннюю базу знаний (Wiki), глобальные системные настройки и логирование. Это фундамент, на котором строятся все остальные бизнес-модули.

## 2. Компоненты (Components)
- **Wiki:** Внутренняя база знаний для сотрудников (инструкции, регламенты, стандарты).
- **Storage:** Управление загрузкой и хранением файлов (макеты, документы, аватары).
- **Core Settings:** Глобальные параметры системы, не относящиеся к брендингу (лимиты, API-ключи, системные флаги).

## 3. Требования (Requirements)
- [x] Полнотекстовый поиск по Wiki.
- [x] Иерархическая структура разделов Wiki.
- [x] Интеграция с S3 (или локальным хранилищем) для файлов.
- [x] Автоматическое создание превью для изображений.
- [ ] Система версионности страниц Wiki.
- [ ] Аудит логов системных событий.

## 4. Техническая реализация (Implementation)
> Стандарт: [[010-Стандарты/Actions|Server Actions v3.0]]

**Файлы:**
- **Схемы БД:**
  - `lib/schema/system.ts` — Системные настройки и глобальное состояние.
  - `lib/schema/wiki.ts` — Структура страниц и категорий Wiki.
  - `lib/schema/storage.ts` — Реестр загруженных файлов и метаданные.
  - `lib/schema/system-fonts.ts` — Реестр доступных системных шрифтов.
- **Интерфейс:**
  - `app/(main)/dashboard/wiki` — Интерфейс базы знаний.
  - `app/(main)/dashboard/settings/system` — Тонкая настройка ядра.
