# 🔒 Безопасность

Защита данных пользователей и инфраструктуры через внедрение строгих стандартов разработки.

## Принципы
- **Безопасные мутации:** Любое изменение данных через Server Actions должно проходить через `createSafeAction`.
- **Защита от CSRF:** Обязательное использование декоратора `withCsrf` для всех экшенов.
- **Схема валидации:** Каждый входной параметр должен быть валидирован через Zod-схему.

## Задачи
- Ревью кода на предмет SQLi (использование параметров в Drizzle).
- Проверка политик доступа (RBAC) в middleware и серверных компонентах.
- Санитайзинг пользовательского ввода перед рендерингом (XSS).
- Мониторинг безопасности зависимостей (npm audit).
