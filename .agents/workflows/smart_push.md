Этот воркфлоу описывает процесс, которому должен следовать Antigravity при задаче "Деплой" или "Пуш в гитхаб".

## Алгоритм работы (CLI Edition)

### 1. Запуск автоматизации
Вместо ручного выполнения git-команд, Antigravity должен использовать скрипт:
```bash
bash ./scripts/smart_push.sh
```

### 2. Что делает скрипт:
- **Локальная проверка**: Запускает `./scripts/predeploy.sh`. Если есть ошибки — Antigravity должен их исправить и перезапустить.
- **Интеллектуальный Пуш**: Если ветка `main` защищена, скрипт автоматически создает временную ветку `push/...`.
- **Авто-PR и Merge**: Через GitHub CLI (`gh`) создается Pull Request и выполняется автоматическое слияние (Merge) с обходом защиты (через `--admin` или `--auto`), если тесты пройдены.
- **Деплой**: Сразу после слияния в `main` GitHub Actions запускает финальный деплой на сервер.

### 3. Завершение
После завершения скрипта, сообщить пользователю итоговый хеш коммита в `main`.
