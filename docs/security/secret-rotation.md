# Ротация AUTH_SECRET

## Что это и когда ротировать

**AUTH_SECRET** — основной криптографический ключ системы, используемый для подписи сессий, JWT и 2FA секретов.

### Когда ротировать:
- **Планово**: каждые 90 дней.
- **При изменениях в команде**: при увольнении сотрудника с доступом к секретам.
- **Экстренно**: при подозрении на компрометацию (утечка в логи, git и т.д.).

## Плановая процедура (без разлогина пользователей)

Механизм ротации позволяет сменить ключ плавно. В течение переходного периода система будет принимать сессии, подписанные как новым, так и старым ключом.

### 1. Генерация нового секрета
Запустите скрипт для автоматической генерации:
```bash
npm run secret:rotate
```
Скрипт выведет новый ключ и инструкции.

### 2. Обновление конфигурации
Обновите файл `.env` (или переменные в Docker/CI):
```bash
# Старый AUTH_SECRET переместите сюда:
AUTH_SECRET_PREVIOUS=старый_секрет

# Новый секрет установите сюда:
AUTH_SECRET=новый_секрет_из_скрипта
```

### 3. Перезагрузка приложения
Перезапустите сервис, чтобы применить новые ключи.
```bash
docker compose up -d --build
# или через PM2/systemd
```

### 4. Завершение ротации (через 7 дней)
Когда большинство активных сессий обновится (пройдёт время `updateAge`), удалите старый секрет:
1. Удалите переменную `AUTH_SECRET_PREVIOUS` из `.env`.
2. Перезапустите приложение.

---

## Экстренная ротация (при компрометации)

Если ключ был украден, необходимо мгновенно инвалидировать все сессии.

### Процедура:
1. Сгенерируйте новый ключ: `npm run secret:rotate`.
2. Установите его в `AUTH_SECRET`.
3. **Оставьте `AUTH_SECRET_PREVIOUS` пустым.**
4. Перезапустите приложение.

**Результат**: Все пользователи будут мгновенно разлогинены и им потребуется повторный вход. Это необходимо для обеспечения безопасности.

---

## Проверка статуса
Статус ротации можно проверить в админ-панели (раздел Безопасность) или через CLI:
```bash
npm run secret:status
```
