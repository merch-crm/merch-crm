# Схема связей (Database ERD)

Ниже представлена высокоуровневая схема связей между основными модулями MerchCRM Unified v3.0.

## 1. Ядро: Заказы и Клиенты

```mermaid
erDiagram
    CLIENTS ||--o{ ORDERS : "размещает"
    ORDERS ||--o{ ORDER_ITEMS : "содержит"
    ORDER_ITEMS }|--|| INVENTORY_ITEMS : "ссылается на товар"
    ORDERS ||--o{ PAYMENTS : "оплата"
    ORDERS ||--o{ ORDER_DESIGN_TASKS : "задачи дизайна"
    CLIENTS ||--o{ CLIENT_CONTACTS : "имеет контакты"
```

## 2. Производство и Склад

```mermaid
erDiagram
    ORDER_ITEMS ||--o{ PRODUCTION_TASKS : "создает задачу"
    PRODUCTION_TASKS }|--|| PRODUCTION_LINES : "выполняется на"
    PRODUCTION_TASKS }|--|| EQUIPMENT : "использует"
    INVENTORY_ITEMS ||--o{ INVENTORY_STOCKS : "хранится в"
    INVENTORY_STOCKS }|--|| STORAGE_LOCATIONS : "находится в"
```

## 3. Персонал и Задачи

```mermaid
erDiagram
    USERS ||--o{ TASKS : "создает/владеет"
    TASKS ||--o{ TASK_ASSIGNEES : "назначена на"
    TASK_ASSIGNEES }|--|| USERS : "исполнитель"
    USERS }|--|| ROLES : "имеет роль"
    USERS }|--|| DEPARTMENTS : "принадлежит отделу"
    USERS ||--o{ WORK_SESSIONS : "учет времени"
```

## Как читать схему
- `||--o{`: Один к многим (обязательно один к нулю или многим).
- `}|--||`: Многие к одному.
- Каждая стрелка отражает связь по `Foreign Key` в схеме Drizzle.
