# 🗄️ Схема Связей БД (ERD)

Ниже представлена высокоуровневая схема связей между основными модулями MerchCRM Unified v3.0.

## 1. Ядро: Заказы и Клиенты

```mermaid
erDiagram
    Клиенты ||--o{ Заказы : "размещает"
    Заказы ||--o{ Элементы_Заказа : "содержит"
    Элементы_Заказа }|--|| Складские_Позиции : "ссылается на товар"
    Заказы ||--o{ Платежи : "оплата"
    Заказы ||--o{ Задачи_Дизайна : "задачи дизайна"
    Клиенты ||--o{ Контакты_Клиента : "имеет контакты"
```

## 2. Производство и Склад

```mermaid
erDiagram
    Элементы_Заказа ||--o{ Производственные_Задачи : "создает задачу"
    Производственные_Задачи }|--|| Линии_Производства : "выполняется на"
    Производственные_Задачи }|--|| Оборудование : "использует"
    Складские_Позиции ||--o{ Остатки_На_Складе : "хранится в"
    Остатки_На_Складе }|--|| Места_Хранения : "находится в"
```

## 3. Персонал и Задачи

```mermaid
erDiagram
    Сотрудники ||--o{ Задачи : "создает/владеет"
    Задачи ||--o{ Исполнители_Задач : "назначена на"
    Исполнители_Задач }|--|| Сотрудники : "исполнитель"
    Сотрудники }|--|| Роли : "имеет роль"
    Сотрудники }|--|| Отделы : "принадлежит отделу"
    Сотрудники ||--o{ Рабочие_Смены : "учет времени"
```

## Как читать схему
- `||--o{`: Один ко многим (обязательно один к нулю или многим).
- `}|--||`: Многие к одному (или обязательно один).
- Каждая стрелка отражает связь по внешнему ключу (`Foreign Key`) в схеме Drizzle ORM.
