# 🛠️ Стек Технологий (MerchCRM)

## 1. Философия Выбора
Технологический стек MerchCRM подобран с упором на **максимальную производительность (Performance)**, **строгую типизацию (End-to-End Type Safety)** и **интегрированный Developer Experience (DX)**. Мы используем монорепо-подход на базе Next.js, что позволяет бесшовно шарить типы между клиентом и сервером.

---

## 2. Глобальная Архитектура (The Stack)

### 🎨 Frontend (Слой Представления)
* **Фреймворк:** [Next.js 15.0+ (App Router)](https://nextjs.org/) — основа нашего приложения. Выбран за встроенную поддержку React Server Components (RSC) и превосходный роутинг.
* **UI Библиотека:** [React 19](https://react.dev/) — используем последние фичи вроде `<form action="...">`, `useOptimistic` и хуков параллельного рендеринга.
* **Стилизация:** [Tailwind CSS v4 (Oxide)](https://tailwindcss.com/blog/tailwindcss-v4) — новый Lightning-быстрый движок без конфигурационных файлов. Обеспечивает строгий консистентный дизайн-код (см. [[010-Стандарты/UI-UX|UX стандарты]]).
* **3D & Canvas:** Fabric.js (для 2D редактора макетов) и Three.js/React Three Fiber (для превью мокапов) (см. [[Turbopack-и-3D-Шейдеры]]).

### ⚙️ Backend & State (Слой Логики)
* **Передача данных:** [React Server Actions](https://react.dev/reference/react/useActionState) — отказ от классических REST API в пользу типизированных вызовов серверов напрямую из компонентов.
* **Аутентификация:** [Better Auth](https://better-auth.com/) — современное типобезопасное решение для авторизации. Работает на базе Redis для хранения сессий (см. [[Аутентификация-Better-Auth]]).
* **Валидация:** [Zod](https://zod.dev/) — строгая валидация всех входящих данных на клиенте и сервере с выводом TS-типов.

### 🗄️ Хранилище (Слой Данных)
* **Основная БД:** [PostgreSQL](https://www.postgresql.org/) — реляционная СУБД для всех бизнес-сущностей, транзакций и HR-данных.
* **ORM:** [Drizzle ORM](https://orm.drizzle.team/) — легковесная, SQL-подобная, 100% типобезопасная ORM без тяжелого рантайма.
* **Кеширование & Сессии:** [Redis](https://redis.io/) — используется для хранения временных токенов, Rate Limiting и кеширования тяжелых запросов витрины (см. [[Кеширование-через-Redis]]).
* **Медиа-хранилище:** S3-совместимое хранилище (MinIO локально / AWS S3 в проде) для загрузки макетов, фотографий товаров и аватарок ([[Хранилище]]).

### 🧪 Quality Assurance (QA & DevOps)
* **Unit/Integration Тесты:** [Vitest](https://vitest.dev/) — сверхбыстрый runner (работает на Vite), заменяет Jest.
* **E2E Тесты:** [Playwright](https://playwright.dev/) — автоматизация браузерных сценариев и визуальное регрессионное тестирование.
* **Mocking:** MSW (Mock Service Worker) для изоляции тестов от реальной сети.
* **Деплой:** Docker-контейнеризация, деплой на собственный сервер (`89.104.69.25`) (см. [[SSH-Туннель-к-Docker]]).

## 3. Топология Взаимодействия (Topology)

```mermaid
flowchart TD
    %% Узлы
    subgraph Client [Клиент (Браузер)]
        UI[React 19 + Tailwind v4]
        Editor[Fabric.js / Three.js Canvas]
    end

    subgraph Server [Next.js Server (Node.js)]
        RSC[Server Components]
        SA[Server Actions]
        AuthZ[Better Auth]
    end

    subgraph DataBase [Инфраструктура Данных]
        DB[(PostgreSQL)]
        Cache[(Redis)]
        S3[(S3 Storage)]
    end

    %% Связи
    UI -.->|useActionState| SA
    UI -->|Render| RSC
    Editor -->|Upload| S3
    
    SA -->|Validate| AuthZ
    SA -->|Drizzle ORM| DB
    SA -.->|Get/Set Cache| Cache
    AuthZ -->|Manage Sessions| Cache
```
