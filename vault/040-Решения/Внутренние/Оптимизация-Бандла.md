# Оптимизация бандла

## Проблема
Проект использует исключительно библиотеку **lucide-react** для иконок. Остальные библиотеки (phosphor-icons, tabler-icons, react-icons) были полностью удалены в ходе оптимизации бандла.

## Решение
Используется `optimizePackageImports` в `next.config.ts`:

```typescript
experimental: {
    optimizePackageImports: [
        "lucide-react",
        "date-fns",
        "recharts",
        "framer-motion",
    ],
}
```

### Как это работает
Next.js перехватывает бочковые импорты (barrel imports) вида:
```typescript
import { User, Settings, Home } from "lucide-react";
```
И автоматически преобразует их в точечные (tree-shakeable):
```typescript
import User from "lucide-react/dist/esm/icons/user";
import Settings from "lucide-react/dist/esm/icons/settings";
```

### Дополнительные оптимизации
- `@next/bundle-analyzer` включается через `ANALYZE=true npm run build`
- `removeConsole: true` в production — удаляет все `console.log`
- `output: "standalone"` — минимальная Docker-сборка
- Оптимизация изображений: WebP формат, агрессивный кеш (31536000 сек в production)

## Файлы
- `next.config.ts` — вся конфигурация оптимизации
