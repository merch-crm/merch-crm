# SSH-Туннель к Docker-сервисам

## Проблема
Локальная разработка требует подключения к PostgreSQL и Redis, которые работают в Docker-контейнерах на удалённом сервере `89.104.69.25`. Прямой доступ невозможен — порты не открыты наружу.

## Решение
Скрипт `scripts/setup-ssh-tunnel.mjs` создаёт **адаптивный SSH-туннель** через `autossh`, автоматически пробрасывая три порта:

```
Локальный DB порт → 127.0.0.1:5432 (Postgres на хосте сервера)
Локальный Redis порт → <Container IP>:6379 (прямо в Docker-контейнер)
Локальный 1984 → 127.0.0.1:1984 (go2rtc, опционально)
```

### Ключевые особенности
1. **Автоопределение IP контейнера Redis** — скрипт по SSH выполняет `docker inspect` и получает внутренний IP контейнера, потому что Redis не проброшен на хост.
2. **Автоматическая очистка портов** — перед запуском убивает все процессы на целевых портах через `lsof`.
3. **Устойчивость** — `autossh` с `ServerAliveInterval=30` и `ServerAliveCountMax=3` переподключает туннель при обрыве.
4. **Фоновый режим** — туннель запускается как detached-процесс и не блокирует терминал.

### Конфигурация
Порты берутся из `.env.local`:
- `DATABASE_URL` → парсится порт БД (regex `/:(\d+)/`)
- `REDIS_PORT` → порт Redis (по умолчанию 6379)

## Файлы
- `scripts/setup-ssh-tunnel.mjs` — основной скрипт
- `scripts/dev.sh` — обёртка, запускающая туннель + `npm run dev`

## Известные проблемы
- При смене IP контейнера Redis нужно перезапустить туннель
- SSH-ключ `~/.ssh/antigravity_key` должен быть настроен заранее
