# Локальная разработка (Setup Guide)

Для быстрой настройки локального окружения в MerchCRM используется Antigravity-воркфлоу `/start_dev_session`.

## 1. Требования
- **Node.js**: v20+.
- **npm**: v10+.
- **SSH-ключ**: `~/.ssh/antigravity_key` должен быть добавлен на сервер `89.104.69.25` (root).
- **autossh**: Для автоматического восстановления SSH-туннеля (`brew install autossh`).
- **GitHub CLI**: `gh` для автоматизации PR (`brew install gh`).

## 2. Быстрый старт

### Через Antigravity (рекомендуемый)
```
@[/start_dev_session]
```

### Ручной запуск
```bash
# 1. Очистка старых процессов
killall autossh ssh rsync 2>/dev/null
lsof -ti:5432 | xargs kill -9 2>/dev/null
lsof -ti:6379 | xargs kill -9 2>/dev/null

# 2. Получить IP Redis-контейнера
REDIS_IP=$(ssh -i ~/.ssh/antigravity_key root@89.104.69.25 \
  "docker inspect -f '{{range.NetworkSettings.Networks}}{{.IPAddress}}{{end}}' merch-crm-redis")

# 3. Поднять SSH-туннель (PostgreSQL + Redis)
AUTOSSH_GATETIME=0 autossh -M 0 -i ~/.ssh/antigravity_key \
  -o "ServerAliveInterval=30" -o "ServerAliveCountMax=3" \
  -o "ExitOnForwardFailure=yes" -f -N \
  -L 5432:127.0.0.1:5432 \
  -L 6379:${REDIS_IP}:6379 \
  root@89.104.69.25

# 4. Синхронизация пароля БД
ssh -i ~/.ssh/antigravity_key root@89.104.69.25 \
  "docker exec merch-crm-db psql -U postgres -c \"ALTER USER postgres WITH PASSWORD '<пароль_из_.env.local>';\""

# 5. Синхронизация файлов
rsync -az -e "ssh -i ~/.ssh/antigravity_key" \
  root@89.104.69.25:/root/merch-crm/local-storage/ ./local-storage/

# 6. Запуск dev-сервера
npm run dev
```

## 3. Работа с базой данных (Drizzle)

Все изменения схемы осуществляются через Drizzle Kit.

- **Генерация миграций**: `npx drizzle-kit generate`
- **Просмотр БД (Drizzle Studio)**: `npx drizzle-kit studio`
- **Проверка изменений**: `npx drizzle-kit check`

> [!WARNING]
> Локальная среда подключается к **продакшн-базе данных** через SSH-туннель. Будьте предельно осторожны при удалении данных или изменении схемы.

## 4. Smart Push (CI/CD)

Для деплоя используйте воркфлоу:
```
@[/smart_push]
```

### Что делает скрипт `scripts/smart_push.sh`:
1. **Predeploy checks**: TypeScript, ESLint, Vitest (377 тестов), Next.js build, Drizzle schema check.
2. **Obsidian sync**: Синхронизация документации модулей.
3. **Commit**: Создание коммита с сообщением.
4. **Branch**: Создание `push/recovery-*` ветки (т.к. `main` защищена).
5. **PR + Merge**: Автоматическое создание PR через `gh` и squash merge через `--admin`.
6. **Deploy**: GitHub Actions автоматически запускает деплой после merge в `main`.

## 5. Troubleshooting

### SSH-туннель зависает / не подключается
```bash
# Убить все SSH-процессы
killall autossh ssh 2>/dev/null
lsof -ti:5432 | xargs kill -9 2>/dev/null

# Проверить сервер
ping -c 3 89.104.69.25
ssh -i ~/.ssh/antigravity_key -o ConnectTimeout=10 root@89.104.69.25 "echo OK"
```
Если SSH не проходит — перезагрузить VPS через панель хостинга.

### Пароль БД не совпадает
Проверьте пароль в `.env.local` (`DATABASE_URL`) и синхронизируйте:
```bash
ssh -i ~/.ssh/antigravity_key root@89.104.69.25 \
  "docker exec merch-crm-db psql -U postgres -c \"ALTER USER postgres WITH PASSWORD '<пароль>';\""
```

## 6. Переменные окружения (.env.local)
Минимальный набор для работы локально:
```bash
DATABASE_URL=postgresql://postgres:<password>@localhost:5432/postgres?sslmode=disable
TEST_DATABASE_URL=postgresql://postgres:<password>@127.0.0.1:5432/merch_crm_test?sslmode=disable
BETTER_AUTH_SECRET=***
RESEND_API_KEY=***
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
```
