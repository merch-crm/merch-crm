# Локальная разработка (Setup Guide)

Для быстрой настройки локального окружения в MerchCRM используется автоматизированный скрипт `scripts/dev.sh`.

## 1. Требования
- **Node.js**: v20+.
- **npm**: v10+.
- **SSH-ключ**: Должен быть добавлен на сервер `89.104.69.25` (root).

## 2. Быстрый старт
Выполните команду в корне проекта:
```bash
bash scripts/dev.sh
```

### Что делает скрипт:
1.  **SSH-туннель**: Прокидывает порт `5432` с удаленного сервера на локальную машину через `setup-ssh-tunnel.mjs`.
2.  **DB Password Sync**: Синхронизирует пароль пользователя `postgres` в Docker-контейнере на сервере.
3.  **DB Check**: Проверяет доступность базы данных.
4.  **Port Cleanup**: Освобождает порт `3000` (убивает зависшие процессы Node).
5.  **Start**: Запускает `npm run dev`.

## 3. Работа с базой данных (Drizzle)

Все изменения схемы осуществляются через Drizzle Kit.

- **Генерация миграций**: `npx drizzle-kit generate`
- **Просмотр БД (Drizzle Studio)**: `npx drizzle-kit studio`
- **Пуш изменений (Dev)**: `npx drizzle-kit push`

> [!WARNING]
> Локальная среда по умолчанию подключается к **продакшн-базе данных** через SSH-туннель. Будьте предельно осторожны при удалении данных или изменении схемы.

## 4. Переменные окружения (.env)
Минимальный набор для работы локально:
```bash
DATABASE_URL=postgres://postgres:5738870192e24949b02a700547743048@localhost:5432/postgres
BETTER_AUTH_SECRET=***
RESEND_API_KEY=***
```

## 5. Полезные команды
- **Тесты**: `npm test`
- **Линтер**: `npm run lint`
- **Сборка**: `npm run build`
