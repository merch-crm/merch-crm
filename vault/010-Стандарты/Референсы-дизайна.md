# Референсы дизайна

## Расположение

Папка `vault/Референсы-Дизайн/` — символическая ссылка на локальное хранилище. В Git не попадает (`.gitignore`). Доступна только на машине разработчика. Все файлы лежат в одной плоской папке без подкатегорий.

## Правила именования файлов

Формат: `[описание]-[номер].png`

Примеры: `дашборд-тёмный-01.png`, `карточка-с-графиком-03.png`, `необычная-навигация-02.png`.

Допустимые форматы: `.png`, `.jpg`, `.webp`, `.svg`, `.gif`.

## Роль референсов

Референсы — это **только вдохновение**. Они нужны чтобы UI не выглядел шаблонно и безлико, а имел характер и визуальную глубину. Агент никогда не копирует референс напрямую. Агент извлекает из референса идеи — нестандартную компоновку, интересный микро-эффект, необычное использование пространства — и адаптирует их строго в рамках стандартов проекта.

## Как агенту работать с референсами

1. Сначала прочитай [[UX-дизайн]] и [[Компоненты-UI]] — это закон
2. Затем просмотри референсы в папке — это вдохновение
3. Из референса бери только **идею**: композицию, ритм, акценты, настроение
4. Реализуй идею используя **только** компоненты, цвета, отступы и типографику из стандартов vault
5. Если хочется отступить от стандарта ради идеи из референса — не отступай. Стандарт важнее

## Приоритет (строгий)

1. [[UX-дизайн]] — **абсолютный приоритет**, нарушать нельзя
2. [[Компоненты-UI]] — **обязательная база**, использовать существующие компоненты
3. Tailwind CSS v4 tokens — **единственный источник** цветов, размеров, отступов
4. Референсы — **только вдохновение**, ни один визуальный параметр не берётся из референса напрямую
