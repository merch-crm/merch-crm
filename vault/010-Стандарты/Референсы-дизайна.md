# Референсы дизайна

Дизайн-система MerchCRM опирается на **три источника**. Каждый играет свою роль и используется в определённом порядке.

---

## Источник 1 — UI Kit (`/ui-kit`)

**Роль: Закон. Обязательная база.**

Полноценная библиотека из **96 компонентов** в **20 категориях**: карточки, кнопки, поля ввода, бейджи, переключатели, вкладки, таблицы, модальные окна, системные уведомления, тултипы, дропдауны, поповеры, слайдеры, аккордеоны, радио-кнопки, аватары, прогресс, скелетоны, статистика.

- **Путь в CRM**: `/ui-kit`
- **Код**: `app/ui-kit/`
- **Содержимое**: Каждый компонент показан в 3 viewport'ах (Desktop 1200px, Tablet 768px, Mobile 375px) с CSS-свойствами
- **Дополнительные подстраницы**: Хэдеры (10 вариантов), Футеры (10), Категории (10), Модальные окна, Ошибки (5), Дропдауны Kit (13), Пагинация (30)
- **Как использовать**: Все новые интерфейсы строятся строго на этих компонентах. Стили, размеры, отступы, поведение — только из UI Kit

---

## Источник 2 — Страница «Референсы» (`/dashboard/references`)

**Роль: Рабочий образец. Эталон стиля.**

Коллекция ~20 премиальных интерактивных React-компонентов, построенных на стандартах проекта. Сгруппированы по категориям:

- **Логистика и доставка** — виджеты отслеживания, транспортные статусы
- **Карточки** — варианты карточек контента и товаров
- **Виджеты** — графики активности, интерактивные блоки
- **Календари** — варианты выбора дат и планирования
- **Интерфейс** — кнопки, переключатели, формы в стиле Lumin-Apple
- **Выпадающие списки** — паттерны навигации и фильтрации
- **Искусственный интеллект** — AI-компоненты

**Как использовать**:

- **Путь в CRM**: `/dashboard/references`
- **Код**: `app/(main)/dashboard/references/`
- Паттерны компоновки, стиль анимаций, типографика, цвет — всё это ориентир
- При создании нового экрана — сверять с этими компонентами для сохранения визуальной консистентности
- Можно переиспользовать конкретные решения (hover-эффекты, скругления, тени), адаптируя под контекст

---

## Источник 3 — Локальные изображения (`vault/Референсы-Дизайн/`)

**Роль: Вдохновение. Только идеи.**

Папка с ~96 файлами (.jpg/.png/.webp) — скриншоты из лучших продуктов и интерфейсов. Символическая ссылка на локальное хранилище. В Git не попадает (`.gitignore`). Доступна только на машине разработчика.

- **Путь**: `vault/Референсы-Дизайн/`  
- **Оригинал**: `/Users/leonidmolchanov/Desktop/WORK/Project/Merch-CRM/Дизайн Референсы`
- **Допустимые форматы**: `.png`, `.jpg`, `.webp`, `.svg`, `.gif`
- **Как использовать**: Из картинок берётся только **идея** — нестандартная компоновка, микро-эффект, необычное использование пространства, настроение. Ни один визуальный параметр не берётся напрямую

> [!warning] Внимание
> При смене машины или окружения необходимо проверять наличие симлинка `vault/Референсы-Дизайн/`.

## Приоритет при разработке UI (строгий)

| Приоритет | Источник | Правило |
|---|---|---|
| 🥇 P1 | [[UX-дизайн]] | Абсолютный приоритет. Нарушать **нельзя** |
| 🥈 P2 | UI Kit (`/ui-kit`) | Обязательная база. Использовать **существующие** компоненты |
| 🥉 P3 | Страница «Референсы» (`/dashboard/references`) | Рабочий образец. Сверять стиль и паттерны |
| 4️⃣ P4 | Tailwind CSS v4 tokens | Единственный источник цветов, размеров, отступов |
| 5️⃣ P5 | Локальные картинки (`vault/Референсы-Дизайн/`) | Только вдохновение. Ничего не копировать напрямую |

---

## Как агенту работать с дизайном (пошагово)

1. Прочитай [[UX-дизайн]] и [[Компоненты-UI]] — это закон
2. Открой `/ui-kit` в браузере — выбери подходящие компоненты
3. Просмотри `/dashboard/references` — сверь стиль и паттерны
4. Просмотри картинки в `vault/Референсы-Дизайн/` — ищи идеи
5. Из картинок бери только **идею**: композицию, ритм, акценты, настроение
6. Реализуй используя **только** компоненты из UI Kit + цвета/отступы из Tailwind tokens
7. Если хочется отступить от стандарта ради идеи из референса — **не отступай**
