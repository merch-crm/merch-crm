# 🛡️ Безопасность MerchCRM

Основные принципы и технические регламенты обеспечения безопасности системы.

## 1. Защита Server Actions
Все мутации данных через Server Actions **обязаны** использовать многоуровневую защиту:
- **Валидация**: Использование `zod` схем для строгой типизации входящих данных.
- **CSRF**: Обязательное использование `withCsrf` для всех действий, изменяющих состояние БД.
- **RBAC**: Проверка ролей через middleware или системные промпты (для AI).

```typescript
// Пример защищенного экшена
export const updateOrder = createSafeAction(OrderSchema, async (data) => {
  return await withAuth(async (session) => {
    // Внутренняя проверка прав и CSRF токена
    return await orderService.update(data, session.user.id);
  });
});
```

## 2. Работа с БД (Drizzle ORM)
- **SQLi**: СТРОГО запрещено использование конкатенации строк. Только `sql` теги Drizzle.
- **Транзакции**: Все связанные изменения (заказ + склад + финансы) должны быть атомарными.

## 3. Файловое хранилище
- **Image-Src**: Поддержка S3 хранилища (Reg.ru) и локального `local-storage/`.
- **WebP**: Автоматическая конвертация при загрузке для снижения риска внедрения вредоносных данных в метаданные.
