# Обработка ошибок (Unified v3.0)

В MerchCRM используется централизованная система обработки ошибок, обеспечивающая безопасность данных и качественный фидбек пользователю.

## 1. Бэкенд: Классы ошибок

Все ошибки должны наследоваться от базового класса `AppError` (находится в `@/lib/errors`).

- **`AppError`**: Базовый класс для всех ожидаемых исключений. Принимает `message` и `code` (ErrorCode).
- **`ValidationError`**: Специальный класс для ошибок Zod-валидации. Дополнительно хранит массив `issues`.

### Рекомендуемые коды ошибок (ErrorCode):
- `BAD_REQUEST`: Ошибка во входных данных.
- `UNAUTHORIZED`: Пользователь не вошел в систему.
- `FORBIDDEN`: Недостаточно прав для действия.
- `NOT_FOUND`: Ресурс не существует.
- `VALIDATION_ERROR`: Ошибка валидации через Zod.
- `INTERNAL_ERROR`: Непредвиденная ошибка на сервере.

## 2. Server Actions и обработка

Оболочка `createSafeAction` автоматически перехватывает ошибки.

- Если внутри `handler` выброшена любая ошибка, она попадает в `handleActionError`.
- **Логирование**: Непредвиденные ошибки (не `AppError`) автоматически логируются в базу через `@/lib/error-logger`.

```typescript
// Пример выброса ошибки
if (!product) {
  throw new AppError("Товар не найден", "NOT_FOUND");
}
```

## 3. Фронтенд: Уведомления

Для информирования пользователя используется библиотека **Sonner**.

### Правила вывода:
- Ошибка должна быть конкретной и на русском языке.
- Используйте `toast.error(message)` для вывода ошибок из `ActionResult`.
- Для критических ошибок с объяснением можно отображать краткое сообщение, а детали (issues) выводить списком.

```typescript
const result = await someAction();
if (!result.success) {
  toast.error(result.error, {
      description: result.issues?.join(', ')
  });
}
```

## 4. Глобальные Crash-отчеты
Все необработанные ошибки (Hydration, Render) перехватываются глобальным компонентом `ErrorBoundry`, который отображает дружелюбное окно восстановления.
