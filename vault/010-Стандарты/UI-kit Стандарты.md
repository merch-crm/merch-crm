# 📜 MerchCRM UI Kit: Протокол и Чеклист

Этот документ объединяет правила проектирования и регламент внедрения компонентов в проект.

---

## 🏷 Именование компонентов (Naming)
- [ ] **Стандартизация**: Использовать только функциональные, описательные названия (например, `ColorPicker`, `Select`).
- [ ] **Без «пустых» слов**: Категорически запрещено использовать в названиях слова «Premium», «Pro», «Ultra», «Advanced» и т.п. Компонент должен называться по его функции.
- [ ] **Префикс**: Все новые компоненты системы «Modern Industrial Craft» используют префикс ``.

---

  ## 🔄 Регламент внедрения (Workflow)

  При получении скриншота компонента я действую строго по следующей схеме:

  1.  **Эталон из UI Kit**: Я не пишу код «на глаз» по картинке. Я нахожу этот компонент в директории `app/ui-kit/`. Код из UI Kit — это единственный источник правды для функционала, состояний (hover/active) и анимаций.
  2.  **Обновление или Создание**:
      *   Все компоненты (базовые и кастомные) должны находиться в `@/components/ui/`.
      *   Глубокая вложенность (более 1 уровня подпапки в `ui/`) запрещена.
      *   Если компонент уже существует в `components/ui/` — я обновляю его код до соответствия эталону.
      *   Если компонента нет — я создаю его с нуля в `components/ui/`.
  3.  **Документирование**: В файле UI Kit (директория `app/ui-kit/`) я обязательно заполняю карточку компонента (`ComponentShowcase`):
      *   **Заголовок и Описание**: Только на **русском языке** (никакого английского).
      *   **ImportPath**: Строка импорта с корректным путем.
  4.  **Глобальная замена**: Я нахожу все старые вхождения и самописные реализации этого элемента во всем проекте и заменяю их на новый стандартизированный компонент.
  5.  **Полная зачистка**: После успешной замены я **удаляю файлы старого компонента** и вычищаю неиспользуемый код/экспорты, чтобы в проекте оставался только один верный вариант.

---

## ⚙️ Технические детали
- [ ] **Accessibility**: Кнопкам-триггерам всегда задавать `type="button"`.
- [ ] **Interactivity**: Использовать `ChevronDown` (`opacity-50`, `text-slate-400`) для всех выпадающих списков.
- [ ] **Popover**: Всегда задавать `align="start"` и `sideOffset={8}`.
