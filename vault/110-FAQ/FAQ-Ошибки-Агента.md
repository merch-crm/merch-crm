# 🤖 Исправление ошибок "Agent terminated due to error"

## ❓ Проблема
Агент Antigravity (Gemini/Claude) внезапно прекращает работу с ошибкой "Agent terminated due to error" или "Our servers are experiencing high traffic".

## 🔍 Причины
1. **Переполнение контекста (Context Overflow)**: Если скрипт (например, `npm run dev` или `npm run find`) выдает тысячи строк логов, агент "захлебывается" этим объемом данных.
2. **Лимиты API**: Слишком частые запросы к статусам команд или огромные результаты поиска превышают квоты моделей.
3. **Накопление процессов**: Множество фоновых процессов `node`, `ssh` и `autossh` замедляют систему и вызывают таймауты инструментов агента.

## 🛠️ Решения

### 1. Используйте "тихий" режим разработки
Вместо обычного `npm run dev`, используйте:
```bash
npm run dev:quiet
```
Этот режим фильтрует повторяющиеся запросы к API (нотификации, счетчики), оставляя только важные ошибки и отчеты о компиляции. Это экономит токены и предотвращает падение агента.

### 2. Ограничивайте вывод поиска
Вместо запуска `grep` или `rg` через `run_command`, используйте встроенный инструмент агента `grep_search`. Он автоматически ограничивает вывод до 50 результатов.

### 3. Очистка зависших процессов
Если агент начал тормозить или падать, выполните очистку:
```bash
killall node ssh autossh
```
После этого перезапустите туннель: `npm run dev:ssh`.

### 4. Мониторинг логов
Не заставляйте агента читать весь файл `dev.log`. Просите его проверять только последние 20-50 строк:
`"проверь последние 20 строк dev.log"`
