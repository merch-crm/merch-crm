# 🧪 E2E Тестирование
Комплексная проверка сквозных пользовательских сценариев MerchCRM с участием браузера.

## ⚙️ Правила
- **Сценарии**: Только критические пути (Авторизация, Создание заказа, Оплата).
- **Данные**: Полная изоляция через `playwright.config.ts`.
- **Среда**: Проверка на Staging или локально через `node ./tests/e2e`.

## 🛠 Техническая реализация
- **Инструмент**: Playwright.
- **Директория**: `tests/e2e/*.spec.ts`.
