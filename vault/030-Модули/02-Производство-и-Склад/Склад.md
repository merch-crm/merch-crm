# Склад (Warehouse & Inventory)

Управление товарами, категориями, атрибутами, остатками, перемещениями, линейками продуктов.

---

## 1. Таблица `inventory_categories`

> **Файл:** `lib/schema/warehouse/categories.ts`

Древовидная иерархия категорий (self-reference через `parentId`).

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `name` | `text NOT NULL UNIQUE` | Название |
| `description` | `text` | Описание |
| `icon` | `text` | Иконка |
| `color` | `text` | Цвет |
| `colorMarker` | `text` | Маркер цвета |
| `slug` | `text UNIQUE` | URL-slug |
| `fullPath` | `text` | Полный путь ("Одежда/Футболки/Поло") |
| `prefix` | `text` | Префикс для SKU |
| `parentId` | `uuid FK → self (SET NULL)` | Родительская категория |
| `sortOrder` | `integer DEFAULT 0 NOT NULL` | Порядок |
| `isActive` | `boolean DEFAULT true NOT NULL` | Активна |
| `isSystem` | `boolean DEFAULT false NOT NULL` | Системная |
| `defaultUnit` | `measurementUnitEnum` | Единица измерения по умолчанию |
| `gender` | `text DEFAULT 'masculine' NOT NULL` | Род (для генерации текста) |
| `singularName` | `text` | Единственное число |
| `pluralName` | `text` | Множественное число |
| `showInSku` | `boolean DEFAULT true NOT NULL` | Показывать в SKU |
| `showInName` | `boolean DEFAULT true NOT NULL` | Показывать в имени |
| `level` | `integer DEFAULT 0 NOT NULL` | Уровень вложенности |
| `createdAt` / `updatedAt` | `timestamp NOT NULL` | Даты |

## 2. Таблица `inventory_attribute_types`

> **Файл:** `lib/schema/warehouse/categories.ts`

Справочник типов атрибутов (привязанные к категориям).

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `slug` | `text NOT NULL` | Slug (уникален в пределах категории) |
| `name` | `text NOT NULL` | Название |
| `isSystem` | `boolean DEFAULT false NOT NULL` | Системный |
| `sortOrder` | `integer DEFAULT 0 NOT NULL` | Порядок |
| `categoryId` | `uuid FK → inventoryCategories` | Категория |
| `dataType` | `text DEFAULT 'text' NOT NULL` | Тип данных: text, unit, color |
| `showInSku` | `boolean DEFAULT true NOT NULL` | Показывать в SKU |
| `showInName` | `boolean DEFAULT true NOT NULL` | Показывать в имени |
| `hasColor` | `boolean DEFAULT false NOT NULL` | Есть цвет |
| `hasUnits` | `boolean DEFAULT false NOT NULL` | Есть единицы |
| `hasComposition` | `boolean DEFAULT false NOT NULL` | Есть состав |
| `meta` | `jsonb` | Метаданные |
| `createdAt` / `updatedAt` | `timestamp NOT NULL` | Даты |

### Constraints
- `UNIQUE(slug, categoryId)`

## 3. Таблица `inventory_attributes`

> **Файл:** `lib/schema/warehouse/attributes.ts`

Значения атрибутов (цвета `BLK`, материалы `KUL`, etc.).

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `type` | `text NOT NULL` | Тип (color, material) |
| `name` | `text NOT NULL` | Отображаемое имя |
| `value` | `text NOT NULL` | Код (BLK, KUL) |
| `categoryId` | `uuid FK → inventoryCategories (CASCADE)` | Категория |
| `meta` | `jsonb` | Метаданные `{ hex: '#...' }` |
| `createdAt` / `updatedAt` | `timestamp NOT NULL` | Даты |

### Constraints
- `UNIQUE(type, value, categoryId)`

## 5. Таблица `inventory_items`

> **Файл:** `lib/schema/warehouse/items.ts`

| Поле | Тип | Описание |
|---|---|---|

| `name` | `text NOT NULL` | Полное имя товара |

| `itemType` | `inventoryItemTypeEnum DEFAULT 'clothing' NOT NULL` | Тип |

| `criticalStockThreshold` | `integer DEFAULT 0 NOT NULL` | Критический порог |

| `qualityCode` / `materialCode` / `brandCode` / `attributeCode` / `sizeCode` | `text` | Коды для генерации SKU |
| `image` / `imageBack` / `imageSide` | `text` | Фото (3 ракурса) |
| `thumbnailSettings` | `jsonb` | Настройки миниатюры |

| `costPrice` | `decimal(10,2)` | Себестоимость |

| `isArchived` | `boolean DEFAULT false NOT NULL` | В архиве |
| `archivedAt` / `archivedBy` / `archiveReason` | `...` | Архивация |
| `zeroStockSince` | `timestamp` | Когда обнулился |
| `attributes` | `jsonb DEFAULT '{}'` | Денормализованные атрибуты |
| `imageDetails` | `jsonb DEFAULT '[]'` | Детали изображений |
| `materialComposition` | `jsonb DEFAULT '{}'` | Состав материала |
| `productLineId` | `uuid FK → productLines` | Линейка |
| `baseItemId` | `uuid` | Базовый товар (self-ref, без FK!) |
| `printDesignId` | `uuid FK → printDesigns` | Дизайн принта |
| `printVersionId` | `uuid FK → printDesignVersions` | Версия дизайна |
| `createdBy` | `uuid FK → users` | Создатель |
| `createdAt` / `updatedAt` | `timestamp NOT NULL` | Даты |

### Self-reference
`baseItem → inventoryItems (baseToFinished)` — базовый товар → готовое изделие с принтом.

## 10. Таблица `product_lines`

> **Файл:** `lib/schema/product-lines.ts`

Линейки продуктов (base / finished). Finished = base + коллекция принтов.

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор (не defaultRandom!) |
| `name` | `varchar(255) NOT NULL` | Название |
| `slug` | `varchar(255) NOT NULL` | Slug |
| `type` | `productLineTypeEnum NOT NULL` | base / finished |
| `categoryId` | `uuid FK → inventoryCategories NOT NULL` | Категория |
| `printCollectionId` | `uuid FK → printCollections` | Коллекция принтов |
| `baseLineId` | `uuid` | Базовая линейка (self-ref, без FK!) |
| `commonAttributes` | `jsonb DEFAULT {}` | Общие атрибуты (`Record<string, string>`) |
| `description` | `text` | Описание |
| `image` | `varchar(500)` | Обложка |
| `isActive` | `boolean DEFAULT true NOT NULL` | Активна |
| `sortOrder` | `integer DEFAULT 0 NOT NULL` | Порядок |
| `createdBy` | `uuid FK → users NOT NULL` | Создатель |
| `createdAt` / `updatedAt` | `timestamp NOT NULL` | Даты |

### Локальный enum
`productLineTypeEnum` → `base`, `finished`
