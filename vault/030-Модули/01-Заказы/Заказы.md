# 📦 Модуль: Заказы (Orders)

Центральный модуль системы, управляющий жизненным циклом заказов от поступления до отгрузки и архивации.

## 🗄️ Структура данных (Schema)

### 1. Таблица `orders`
Основная сущность заказа.
- **ID**: UUID (primary key)
- **Статус**: `order_status` (Enum: `new`, `design`, `production`, `done`, `shipped`, `cancelled`, `completed`, `archived`)
- **Финансы**: 
    - `total_amount`: Полная стоимость (decimal)
    - `paid_amount`: Оплаченная сумма
    - `discount_amount`: Сумма скидки
- **Сроки**: `deadline` (timestamp), `priority` (Enum: `low`, `normal`, `high`, `urgent`)
- **Связи**: 
    - `clientId` -> `clients.id`
    - `managerId` -> `users.id` (Ответственный менеджер)

### 2. Таблица `order_items`
Позиции внутри заказа.
- **Связь**: `orderId` -> `orders.id`
- **Номенклатура**: `inventoryId` -> `inventory_items.id`
- **Этапы производства**: 
    - `stage_prep_status` (Подготовка)
    - `stage_print_status` (Печать)
    - `stage_application_status` (Нанесение)
    - `stage_packaging_status` (Упаковка)

## ⚙️ Бизнес-логика

### Жизненный цикл статусов
1. **new**: Новый заказ, созданный менеджером.
2. **design**: Передан дизайнеру для подготовки макетов.
3. **production**: В очереди на производство или в процессе печати/нанесения.
4. **done**: Производство завершено, ожидает отгрузки.
5. **shipped**: Заказ передан в службу доставки или выдан клиенту.
6. **completed**: Финальный статус после подтверждения получения.
7. **archived**: Скрыт из активных списков, хранится для истории.

### Логика расчета стоимости
- Итоговая сумма (`total_amount`) рассчитывается как сумма всех `order_items`.
- При применении промокода (`promocodeId`) сумма пересчитывается с учетом скидки.
- Оплата отслеживается через `paymentStatusEnum` (`unpaid`, `partial`, `paid`).

## 🔗 Интеграции
- **Клиенты**: Привязка заказа к истории клиента для CRM-аналитики.
- **Производство**: Автоматическое создание задач для печатников при переходе в статус `production`.
- **Склад**: Резервирование остатков (`inventory_items`) при создании заказа.
