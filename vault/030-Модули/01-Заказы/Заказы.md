# Заказы (Orders)

Центральный модуль CRM: управление заказами, позициями, вложениями.

---

## 1. Таблица `orders`

> **Файл:** `lib/schema/orders.ts`

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `clientId` | `uuid FK → clients (CASCADE on update)` | Клиент |
| `clientName` | `text` | Денормализованное имя клиента |
| `status` | `orderStatusEnum DEFAULT 'new' NOT NULL` | Статус |
| `totalAmount` | `decimal(10,2) DEFAULT '0' NOT NULL` | Сумма заказа |
| `paidAmount` | `decimal(10,2) DEFAULT '0'` | Оплачено |
| `paymentStatus` | `paymentStatusEnum DEFAULT 'unpaid' NOT NULL` | Статус оплаты |
| `deliveryStatus` | `deliveryStatusEnum DEFAULT 'not_started' NOT NULL` | Статус доставки |
| `shippingAddress` | `text` | Адрес доставки |
| `deliveryMethod` | `text` | Способ доставки |
| `deliveryTrackingNumber` | `text` | Трек-номер |
| `estimatedDeliveryDate` | `timestamp` | Планируемая дата доставки |
| `actualDeliveryDate` | `timestamp` | Фактическая дата доставки |
| `comments` | `text` | Комментарии |
| `managerId` | `uuid FK → users` | Менеджер |
| `createdBy` | `uuid FK → users` | Кто создал |
| `source` | `text` | Источник заказа |
| `deadline` | `timestamp` | Дедлайн |
| `priority` | `text DEFAULT 'normal' NOT NULL` | Приоритет |
| `discountAmount` | `decimal(10,2) DEFAULT '0'` | Скидка |
| `promocodeId` | `uuid FK → promocodes` | Промокод |
| `externalOrderNumber` | `text` | Внешний номер заказа |
| `orderNumber` | `text UNIQUE` | Номер заказа (MRH-XXXX) |
| `isUrgent` | `boolean DEFAULT false NOT NULL` | Срочный |
| `cancelReason` | `text` | Причина отмены |
| `category` | `orderCategoryEnum DEFAULT 'other' NOT NULL` | Категория |
| `isArchived` | `boolean DEFAULT false NOT NULL` | В архиве |
| `archivedAt` | `timestamp` | Дата архивации |
| `archivedBy` | `uuid FK → users` | Кто архивировал |
| `createdAt` / `updatedAt` | `timestamp NOT NULL` | Даты |

### Индексы
- `orders_client_idx`, `orders_status_idx`, `orders_manager_idx`, `orders_created_by_idx`
- `orders_created_idx`, `orders_deadline_idx`, `orders_date_idx`
- `orders_status_date_idx` → `(status, createdAt)`
- `orders_archived_by_idx`, `orders_promocode_idx`
- `orders_status_archived_created_idx` → `(status, isArchived, createdAt)`

### Модель статусов

```
new → design → production → done → shipped → completed
                                              ↗
new → cancelled                     done → completed
                                              ↗
                            любой → archived
```

### Relations
```
orders → client (one)
orders → manager (one, relation: order_manager)
orders → creator (one, relation: order_creator)
orders → archivedBy (one, relation: order_archiver)
orders → promocode (one)
orders → items (many: orderItems)
orders → attachments (many: orderAttachments)
orders → payments (many)
orders → tasks (many)
```

## 2. Таблица `order_items`

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `orderId` | `uuid FK → orders (CASCADE)` | Заказ |
| `inventoryId` | `uuid FK → inventoryItems` | Товар со склада |
| `description` | `text` | Описание позиции |
| `quantity` | `integer NOT NULL` | Количество |
| `price` | `decimal(10,2) NOT NULL` | Цена за единицу |
| `itemDetails` | `text` | Детали позиции |
| `stagePrepStatus` | `productionStageStatusEnum DEFAULT 'pending' NOT NULL` | Статус: подготовка |
| `stagePrintStatus` | `productionStageStatusEnum DEFAULT 'pending' NOT NULL` | Статус: печать |
| `stageApplicationStatus` | `productionStageStatusEnum DEFAULT 'pending' NOT NULL` | Статус: нанесение |
| `stagePackagingStatus` | `productionStageStatusEnum DEFAULT 'pending' NOT NULL` | Статус: упаковка |
| `designStatus` | `orderItemDesignStatusEnum DEFAULT 'pending' NOT NULL` | Статус дизайна |
| `applicationTypeId` | `uuid` | Тип нанесения (без FK constraint!) |
| `createdAt` / `updatedAt` | `timestamp NOT NULL` | Даты |

> ⚠️ У каждой позиции 4 стадии производства (`stage*Status`) + статус дизайна. `applicationTypeId` — uuid без явного FK (ссылка на `applicationTypes`). Нет полей `name`, `sku`, `cost`, `mockupUrl`, `mockupStatus`, `printDesignId`.

### Индексы
- `order_items_order_idx`, `order_items_inventory_idx`, `order_items_created_idx`

### Relations
```
orderItems → order (one)
orderItems → inventory (one: inventoryItems)
orderItems → applicationType (one: applicationTypes)
```
