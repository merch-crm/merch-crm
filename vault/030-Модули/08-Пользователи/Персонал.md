# Персонал (Users & Auth)

Управление пользователями, ролями, отделами, сессиями, 2FA и аккаунтами OAuth.

---

## 1. Таблица `departments`

> **Файл:** `lib/schema/users.ts`

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `name` | `text NOT NULL UNIQUE` | Название |
| `description` | `text` | Описание |
| `color` | `text DEFAULT 'indigo'` | Цвет в UI |
| `isActive` | `boolean DEFAULT true NOT NULL` | Активен |
| `isSystem` | `boolean DEFAULT false NOT NULL` | Системный |
| `updatedAt` | `timestamp NOT NULL` | Обновлено |
| `createdAt` | `timestamp NOT NULL` | Создано |

### Relations
- `departments → roles (many)`, `departments → users (many)`

---

## 2. Таблица `roles`

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `name` | `text NOT NULL UNIQUE` | Название роли |
| `isSystem` | `boolean DEFAULT false NOT NULL` | Системная (не удаляемая) |
| `departmentId` | `uuid FK → departments` | Отдел |
| `color` | `text` | Цвет |
| `createdAt` | `timestamp NOT NULL` | Создано |
| `updatedAt` | `timestamp NOT NULL` | Обновлено |
| `permissions` | `jsonb NOT NULL DEFAULT '{}'` | Права доступа |

### Структура `permissions` (JSON)
```typescript
{
  "orders": { "create": true, "read": true, "update": true, "delete": false },
  "clients": { "create": true, "read": true },
  "admin": { "manage_users": false, "manage_roles": false }
}
```

### Relations
- `roles → department (one)`, `roles → users (many)`

## 3. Таблица `users`

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `name` | `text NOT NULL` | Имя |
| `email` | `text NOT NULL UNIQUE` | Email |
| `roleId` | `uuid FK → roles` | Роль |
| `phone` | `text` | Телефон |
| `birthday` | `date` | Дата рождения |
| `image` | `text` | Аватар (Better Auth standard) |
| `avatar` | `text` | Аватар (legacy, обратная совместимость) |
| `telegram` | `text` | Telegram |
| `instagram` | `text` | Instagram |
| `socialMax` | `text` | Social Max (доп. соцсеть) |
| `departmentId` | `uuid FK → departments` | Отдел |
| `lastActiveAt` | `timestamp` | Последняя активность |
| `isSystem` | `boolean DEFAULT false NOT NULL` | Системный |
| `createdAt` | `timestamp NOT NULL` | Создано |
| `updatedAt` | `timestamp NOT NULL` | Обновлено |
| `emailVerified` | `boolean DEFAULT false NOT NULL` | Email подтверждён |
| `twoFactorEnabled` | `boolean DEFAULT false` | 2FA включена |

> ⚠️ **Два поля аватара:** `image` (Better Auth) + `avatar` (legacy). Нет `password`, `salt`, `passwordHash` — пароль хранится в `accounts`.

### Индексы
- `users_role_idx`, `users_department_idx`, `users_phone_idx`, `users_email_idx`, `users_created_idx`

### Relations
```
users → role (one)
users → department (one)
users → managedOrders (many, relation: order_manager)
users → createdOrders (many, relation: order_creator)
users → archivedOrders (many, relation: order_archiver)
users → payments (many)
users → expenses (many)
users → inventoryTransactions (many)
users → inventoryTransfers (many)
users → taskAssignments (many, relation: userAssignments)
users → taskAssignedBy (many, relation: taskAssignedBy)
users → createdTasks (many, relation: createdTasks)
users → taskDelegated (many, relation: taskDelegator)
users → taskOriginalAssignments (many, relation: taskOriginalAssignee)
users → taskWatching (many, relation: taskWatcherUser)
users → taskWatcherAdded (many, relation: taskWatcherAddedBy)
users → managedClients (many)
users → wikiPages (many)
users → sessions (many)
```

## 4. Таблица `sessions`

| Поле | Тип | Описание |
|---|---|---|
| `id` | `text PK` | Session hash (не uuid!) |
| `token` | `text NOT NULL UNIQUE` | Токен Better Auth |
| `userId` | `uuid FK → users (CASCADE) NOT NULL` | Пользователь |
| `userAgent` | `text` | User-Agent |
| `ipAddress` | `text` | IP-адрес |
| `expiresAt` | `timestamp NOT NULL` | Истекает |
| `createdAt` | `timestamp NOT NULL` | Создано |
| `updatedAt` | `timestamp NOT NULL` | Обновлено |

> ⚠️ `id` — **text** (не uuid), потому что используется secure session hash от Better Auth.

---

## 5. Таблица `accounts`

Аккаунты OAuth/credential для Better Auth.

| Поле | Тип | Описание |
|---|---|---|
| `id` | `text PK` | Идентификатор |
| `accountId` | `text NOT NULL` | ID аккаунта |
| `providerId` | `text NOT NULL` | Провайдер (credential, google, github) |
| `userId` | `uuid FK → users (CASCADE) NOT NULL` | Пользователь |
| `accessToken` | `text` | OAuth access token |
| `refreshToken` | `text` | OAuth refresh token |
| `expiresAt` | `timestamp` | Истечение токена |
| `password` | `text` | Хэш пароля (для credential провайдера) |
| `createdAt` / `updatedAt` | `timestamp NOT NULL` | Даты |

> ⚠️ **Пароль хранится здесь** (в `accounts.password`), не в `users`.
