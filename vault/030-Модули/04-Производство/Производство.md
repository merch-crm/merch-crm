# Производство (Production)

Модуль управления производственными процессами: линии, оборудование, задачи, персонал, тайм-трекинг.

---

## 1. Локальные Enums (в production.ts)

### `applicationCategoryEnum`
`print` | `embroidery` | `engraving` | `transfer` | `other`

### `equipmentStatusEnum`
`active` | `maintenance` | `repair` | `inactive`

### `productionTaskStatusEnum`
`pending` | `in_progress` | `paused` | `completed` | `cancelled`

### `productionTaskPriorityEnum`
`low` | `normal` | `high` | `urgent`

---

## 2. Таблица `application_types`

> **Файл:** `lib/schema/production.ts`

Справочник типов нанесения (DTF, сублимация, вышивка, гравировка).

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `name` | `varchar(255) NOT NULL` | Название |
| `slug` | `varchar(255) NOT NULL UNIQUE` | URL-slug |
| `description` | `text` | Описание |
| `category` | `applicationCategoryEnum DEFAULT 'print' NOT NULL` | Категория |
| `icon` | `varchar(100)` | Иконка (Lucide) |
| `color` | `varchar(7)` | HEX цвет UI |
| `minQuantity` | `integer DEFAULT 1` | Мин. тираж |
| `maxColors` | `integer` | Макс. цветов (null = безлимит) |
| `maxPrintArea` | `varchar(50)` | Макс. область печати "30x40cm" |
| `baseCost` | `integer DEFAULT 0` | Базовая стоимость (копейки) |
| `costPerUnit` | `integer DEFAULT 0` | Стоимость за единицу (копейки) |
| `setupCost` | `integer DEFAULT 0` | Стоимость подготовки (копейки) |
| `estimatedTime` | `integer` | Время на единицу (минуты) |
| `setupTime` | `integer` | Время подготовки (минуты) |
| `isActive` | `boolean DEFAULT true NOT NULL` | Активен |
| `sortOrder` | `integer DEFAULT 0 NOT NULL` | Порядок |

| `updatedAt` | `timestamp NOT NULL` | Обновлено |

## 5. Таблица `production_staff`

Отдельная таблица персонала производства (может не иметь учётную запись в `users`).

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `userId` | `uuid FK → users (SET NULL)` | Связь с системным пользователем |
| `name` | `varchar(255) NOT NULL` | ФИО |
| `phone` | `varchar(20)` | Телефон |
| `email` | `varchar(255)` | Email |
| `position` | `varchar(100)` | Должность |
| `specializationIds` | `jsonb (string[])` | Специализации (UUID типов нанесения) |
| `lineIds` | `jsonb (string[])` | Закреплённые линии (UUID) |
| `hourlyRate` | `integer` | Ставка (копейки/час) |
| `isActive` | `boolean DEFAULT true NOT NULL` | Активен |
| `avatarPath` | `text` | Аватар |
| `createdAt` / `updatedAt` | `timestamp NOT NULL` | Даты |

> ⚠️ `userId` — опциональный FK. Сотрудник производства может быть без учётной записи. `assigneeId` в задачах ссылается на `productionStaff`, **НЕ на users**.

## 6. Таблица `production_tasks`

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `number` | `varchar(50) NOT NULL UNIQUE` | Номер задачи |
| `orderId` | `uuid FK → orders (CASCADE) NOT NULL` | Заказ |
| `orderItemId` | `uuid FK → orderItems (CASCADE)` | Позиция заказа |
| `applicationTypeId` | `uuid FK → applicationTypes (SET NULL)` | Тип нанесения |
| `lineId` | `uuid FK → productionLines (SET NULL)` | Линия |
| `assigneeId` | `uuid FK → productionStaff (SET NULL)` | Исполнитель |
| `title` | `varchar(255) NOT NULL` | Заголовок |
| `description` | `text` | Описание |
| `quantity` | `integer DEFAULT 1 NOT NULL` | Количество |
| `completedQuantity` | `integer DEFAULT 0` | Выполнено |
| `defectQuantity` | `integer DEFAULT 0` | Брак |
| `defectCategory` | `varchar(50)` | Категория брака |
| `defectComment` | `text` | Комментарий к браку |
| `status` | `productionTaskStatusEnum DEFAULT 'pending' NOT NULL` | Статус |
| `priority` | `productionTaskPriorityEnum DEFAULT 'normal' NOT NULL` | Приоритет |
| `startDate` | `timestamp` | Начало |
| `dueDate` | `timestamp` | Дедлайн |
| `completedAt` | `timestamp` | Завершено |
| `estimatedTime` | `integer` | Оценка (минуты) |
| `actualTime` | `integer` | Факт (минуты) |
| `designFiles` | `jsonb ({path, name, type}[])` | Файлы макета |
| `notes` | `text` | Примечания |
| `sortOrder` | `integer DEFAULT 0 NOT NULL` | Порядок в очереди |
| `createdBy` | `uuid FK → users NOT NULL` | Кто создал |
| `createdAt` / `updatedAt` | `timestamp NOT NULL` | Даты |

### CHECK constraint
```sql
CHECK (completed_quantity + defect_quantity <= quantity)
```

### Индексы
- `_number_idx`, `_order_id_idx`, `_line_id_idx`, `_assignee_id_idx`, `_status_idx`, `_priority_idx`, `_due_date_idx`, `_sort_order_idx`, `_created_at_idx`
- Partial: `idx_production_tasks_defect_category` WHERE `defect_quantity > 0`

## 7. Таблица `production_logs`

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `taskId` | `uuid FK → productionTasks (CASCADE)` | Задача |
| `event` | `varchar(50) NOT NULL` | Тип события (started, paused, resumed, completed, quantity_updated, assigned) |
| `details` | `jsonb` | Детали |
| `performedBy` | `uuid FK → users (SET NULL)` | Кто выполнил |
| `createdAt` | `timestamp NOT NULL` | Дата |

> ⚠️ Поле называется `event` (не `action`), `performedBy` (не `userId`).

## 9. Таблица `user_application_types`

Связующая таблица: какие типы нанесения может выполнять пользователь.

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `userId` | `uuid FK → users (CASCADE) NOT NULL` | Пользователь |
| `applicationTypeId` | `uuid FK → applicationTypes (CASCADE) NOT NULL` | Тип нанесения |
| `createdAt` | `timestamp NOT NULL` | Создано |

### Constraints
- `user_app_types_unique_idx` — `UNIQUE(userId, applicationTypeId)`

## 10. Связи (Relations)

```
productionLines → applicationTypes (one)
productionLines → productionTasks (many)
productionStaff → users (one, optional)
productionStaff → productionTasks (many)
productionTasks → orders, orderItems, applicationTypes, productionLines, productionStaff, users
productionTasks → productionLogs (many), productionTimeLogs (many)
applicationTypes → userApplicationTypes (many)
```
