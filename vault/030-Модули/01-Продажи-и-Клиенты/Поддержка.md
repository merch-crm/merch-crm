Внутренняя тикет-система и база знаний.

---

|---|---|

| `ticketPriorityEnum` | `low`, `normal`, `high`, `critical` |
| `ticketCategoryEnum` | `equipment`, `materials`, `design`, `other` |

---

## 2. Таблица `tickets`

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `number` | `varchar(50) NOT NULL UNIQUE` | Номер тикета |
| `createdById` | `uuid FK → users NOT NULL` | Кто создал |
| `title` | `text NOT NULL` | Заголовок |
| `description` | `text` | Описание проблемы |
| `category` | `ticketCategoryEnum DEFAULT 'other' NOT NULL` | Категория |
| `status` | `ticketStatusEnum DEFAULT 'open' NOT NULL` | Статус |
| `priority` | `ticketPriorityEnum DEFAULT 'normal' NOT NULL` | Приоритет |
| `orderId` | `uuid FK → orders (SET NULL)` | Привязка к заказу |
| `orderItemId` | `uuid FK → orderItems (SET NULL)` | Привязка к позиции |
| `assigneeId` | `uuid FK → users` | Ответственный |
| `resolution` | `text` | Решение |
| `resolvedAt` | `timestamp` | Дата решения |
| `createdAt` / `updatedAt` | `timestamp NOT NULL` | Даты |

> ⚠️ **Нет полей** `type`, `severity`, `attachments`, `comments`, `estimatedResolutionTime`, `slaDeadline`, `closedAt`, `closedById`, `relatedTicketId`.

### Relations
```
tickets → creator (one, relation: ticket_creator)
tickets → assignee (one, relation: ticket_assignee)
tickets → order (one)
```

## 3. Таблица `wiki_folders`

> **Файл:** `lib/schema/wiki.ts`

Древовидная иерархия папок (self-reference).

| Поле | Тип | Описание |
|---|---|---|
| `id` | `uuid PK` | Идентификатор |
| `name` | `text NOT NULL` | Название |
| `parentId` | `uuid FK → self` | Родительская папка |
| `sortOrder` | `integer DEFAULT 0 NOT NULL` | Порядок |
| `createdAt` | `timestamp NOT NULL` | Создано |

> ⚠️ **Нет** `updatedAt`, `icon`, `color`, `isActive`, `createdById`.
