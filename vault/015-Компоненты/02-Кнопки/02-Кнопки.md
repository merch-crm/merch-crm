# 02. Кнопки

Система кнопок MerchCRM основана на 6 семантических цветах и 4 визуальных стилях. Это основной интерактивный элемент системы.

## Навигация

- [[Button]] — базовый компонент.
- [[SubmitButton]] — кнопка для отправки форм с состоянием загрузки.

## Цветовая палитра

| Цвет | Назначение | Токен (Solid) |
|---|---|---|
| `black` | Основное действие (Midnight) | `#020617` (Slate 950) |
| `red` | Опасные / Деструктивные (Rose) | `#E11D48` (Rose 600) |
| `yellow` | Предупреждения (Amber) | `#F59E0B` (Amber 500) |
| `purple` | Брендовые / Акцентные (Primary) | `#5D00FF` (Primary) |
| `green` | Успех / Позитивные (Emerald) | `#10B981` (Emerald 600) |
| `gray` | Вторичные / Нейтральные (Slate) | `#F1F5F9` (Slate 500/100) |

## Стили (Variants)

1. **Solid**: Основной стиль с заливкой.
2. **Outline**: Стиль с контуром для вторичных действий того же цвета.
3. **Ghost**: Минималистичный стиль без фона (фон появляется при наведении).
4. **Link**: Текстовая ссылка (только для цвета `gray`).

## Правила использования (Usage Rules)

Для поддержания визуальной целостности и предсказуемости интерфейса следуйте этим правилам:

### 1. Основные действия
Используются для главного целевого действия на экране или внутри формы.
- **Стиль**: `color="black"`, `variant="solid"`
- **Где**: "Сохранить", "Создать", "Отправить", "Принять заказ".
- **Ограничение**: Не более одной кнопки `black solid` на экранный блок.

### 2. Вторичные действия
Для действий, которые дополняют основные, но не являются критическими.
- **Стиль**: `color="gray"`, `variant="outline"` или `variant="ghost"`.
- **Где**: "Отмена", "Назад", "Закрыть", "Сбросить фильтры".

### 3. Групповые операции и Инструменты
Для кнопок в тулбарах таблиц или заголовках страниц.
- **Стиль**: `variant="outline"` (если действие активно) или `variant="ghost"` (для инструментов).
- **Размер**: `size="sm"` в таблицах, `size="md"` в заголовках.

### 4. Опасные действия (Destructive)
Используются для действий, которые приводят к удалению данных или необратимым изменениям.
- **Стиль**: `color="red"`, `variant="outline"` (для подтверждения) или `variant="ghost"` (в списках).
- **Важно**: Для окончательного удаления в модальном окне используйте `color="red"`, `variant="solid"`.

### 5. Семантические акценты
- **WhatsApp/Communication**: Используйте `color="green"` только для действий, связанных с завершением процесса или специфическими каналами связи.
- **Архив/Attention**: Используйте `color="yellow"` для перевода в архив или временной блокировки.

## 🚫 Запрещено (Strict Deny)

1. **Ручные оверрайды высоты**: Не используйте `h-11`, `h-12` и т.д. Используйте `size`.
2. **Ручные закругления**: Не используйте `rounded-2xl`, `rounded-xl`. Компонент сам применяет нужные токены.
3. **Прямое использование классов Tailwind**: Избегайте использования классов `bg-primary`, `bg-slate-900` на кнопках напрямую. Всегда используйте проп `color`.
4. **Тип по умолчанию**: Помните, что `Button` имеет `type="button"`. Внутри форм ОБЯЗАТЕЛЬНО указывайте `type="submit"`.
