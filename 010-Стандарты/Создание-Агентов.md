# 🤖 Стандарт создания AI-агентов

Этот регламент описывает, как правильно добавлять новых "специалистов" в систему MerchCRM, обеспечивая их консистентность и качество работы.

---

## 1. Двухслойная Архитектура

Каждый агент состоит из двух обязательных файлов:

| Слой | Путь | Назначение |
|------|------|------------|
| **System** | `.agents/agents/[name].md` | Внутренние правила, список доступных инструментов и навыков. |
| **Knowledge** | `vault/080-Агенты/##-Категория/[Имя].md` | Описание для пользователя, статус, связи с другими модулями. |

---

## 2. Системные правила (.agents/agents/)

Файл должен содержать четкий YAML frontmatter:

```yaml
---
name: backend-specialist
description: Краткое описание для маршрутизатора
tools: Read, Grep, Edit, ...
skills: clean-code, api-patterns, ...
model: inherit # или конкретная модель
---
```

**Основные разделы:**
1. **Quick Navigation**: Ссылки на секции файла.
2. **Runtime Capability Check**: Что агент должен проверить перед началом работы (например, `ARCHITECTURE.md`).
3. **Domain Boundaries**: Список того, что агенту РАЗРЕШЕНО и ЗАПРЕЩЕНО делать.
4. **Invocation Protocol**: Как именно агент должен использовать инструменты.

---

## 3. Документация в Obsidian (080-Агенты/)

Используйте шаблон `[[070-Шаблоны/Tpl-Агент]]`. 

**Обязательные элементы:**
- **Статус**: (draft / active / deprecated).
- **Специализация**: Какие задачи решает.
- **Связанные навыки**: Ссылки на `[[.agents/skills/]]`.
- **Локация кода**: К каким папкам в репозитории имеет доступ.

---

## 4. Лингвистический стандарт (Russian Only)

Согласно `[[010-Стандарты/Работа-с-Обсидианом|регламенту]]` и `core-language.md`, агент обязан:
1. Думать (Internal Thought) может на английском для точности.
2. **Отвечать пользователю строго на русском языке**.
3. Вести логи и комментарии в коде на русском (кроме переменных).

---

## 5. Процесс добавления

1. Создать системный файл в `.agents/agents/`.
2. Создать документацию в `vault/080-Агенты/`.
3. Добавить агента в индекс `vault/080-Агенты/AI-Агенты.md`.
4. (Опционально) Добавить уникальные цвета в `.obsidian/graph.json`.
